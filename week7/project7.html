Questions

  1. Why is the JavaScript textContent property safer than innerHTML?
    • The old innerHTML method allowed the actual HTML code to be changed which allowed code to be input and executed by anyone. The textContent property only changes the text stored inside the element which doesn't allow code to be executed.
  2. What is the relationship between an HTML attribute and a corresponding DOM property? (what is the connection to JavaScript)
    • Using the DOM method, HTML attributes are essentially properties of the "document" object. This allows them to be targeted with text like: document.body.
    • The individual properties of these attributes like style or textContent can be targeted like: document.body.style.backgroundColor.
